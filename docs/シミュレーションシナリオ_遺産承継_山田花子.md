# シミュレーションシナリオ：遺産承継（山田 花子）

新機能（口座種別／着手フラグ／着手OKセンター／禁止期間⚡モーダル／W-Check依頼集約／解約書類受領のタスク削除／完了モーダルの提案バッジ）を一通り触るためのケース。手続太郎シナリオとは別軸で作成。

---

## 0. 事案の全体像
- **依頼者**：山田 一郎（長男）。母の遺産整理を一括で依頼（遺産承継）。
- **被相続人**：山田 花子（母）。老衰で死亡。晩年は自宅マンションで一人暮らし＋長男が近所で介護。
- **相続人**：一郎（長男）・二郎（次男）の2名。争いなし。
- **財産規模**：不動産（自宅マンション＋賃貸アパート）＋預貯金3行＋証券1社＋保険＝約1.1億円 → 相続税申告あり（税理士紹介）。
- **分割方針**：自宅マンションは一郎、賃貸アパートは二郎、預金は法定相続分で分ける（現物優先）。
- **特徴**：
  - 賃貸アパートあり → **賃料入金の禁止期間**でゲート
  - 三菱UFJに**普通預金＋定期預金の2口座**あり → 口座種別テスト
  - 転籍1回あり → 追加戸籍請求
  - 生命保険あり → 保険金請求手続き

## このシナリオで検証できる新機能一覧

### タスク・着手フラグ関連
- **着手フラグ列**（案件詳細タスクタブ）で⚡着手OK/⏰受領次第OKトグル
- **着手OKセンター**（右上ボタン）で条件が揃ったタスクを一覧提案（不動産・金融）
- **完了モーダルの💡バッジ**：戸籍完了時に「相続手続き請求：三菱UFJ」等が提案される
- **禁止期間⚡モーダル**：8/1以降に三菱UFJ相続手続き請求の⚡クリック→禁止理由確認モーダル

### 金融資産まわり
- **口座種別**（普通/定期/当座/積立/貯蓄）の列で選択・保存
- **凍結状態バッジ**（🔒凍結済/🔓未凍結）が左端に表示
- **右側W-Check依頼ボタン集約**（凍結確認/残高確定）
- **凍結確認名（confirmed_name）**が管理担当ハンコに表示
- 資料読込タスク完了ゲートで **「W-Check依頼漏れ」プロンプト**（残高確定＋凍結確認）まとめて依頼

### 不動産まわり
- **取得予定資料**の[土地]/[建物]バッジ判別
- 資料の複数選択（item_types）
- 評価額確定W-Check
- 評価額ハンコ（confirmed_name）

### 全体
- 戸籍：追加請求承認フロー
- 確認簿：ハンコ表示・履歴
- 財産目録：確定分のみ集計
- 遺産承継：精算書・指図書・代理支払
- 請求：司法（相続の力）／行政（このシステム）
