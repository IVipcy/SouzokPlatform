# 進捗管理ボード フラグ定義

> 案件の進捗リスクを信号機の色（🔵 青 / 🟡 黄 / 🔴 赤）で表すルール

---

## 1. このボードの目的

「**この案件、期限に間に合いそう？それともヤバい？**」を一目で判定するためのボード。

各案件に色フラグを付け、**赤＝すぐ対応が必要、黄＝早めに手を打つべき、青＝順調** を可視化する。チームリーダー / 個人が、リスクのある案件を早期に発見してリカバリ対応につなげるのが狙い。

---

## 2. 判定の前提

### 各案件には2つの期限がある

| 期限 | 中身 |
|---|---|
| **案件全体の完了予定日** | その案件をいつまでに終わらせるかの全体ゴール |
| **個々のタスクの期限** | 各タスクをいつまでに終わらせるか（タスク作成時に設定） |

### 判定対象になる案件
**当月に完了予定日が設定されている案件** のみが対象。

（完了予定日が未設定の案件、来月以降の案件はこのボードには載らない）

---

## 3. フラグの定義

| フラグ | 条件 | 色の意味 |
|:---:|---|---|
| 🔴 **赤** | (a) 案件の完了予定日 ＜ 今日（**既に超過**）<br>または<br>(b) **期限超過タスクが 2件以上** ある | **緊急**：早急なリカバリサポートが必要 |
| 🟡 **黄** | (c) **期限超過タスクが 1件** ある<br>または<br>(d) **期限が3日以内**の未完タスクが**2件以上**ある | **注意**：早めに手を打たないと深刻化する |
| 🔵 **青** | 上記のどれにも当てはまらない | **順調**：予定通り完了できそう |

### 判定優先順位
赤の条件 > 黄の条件 > 青、の順で判定する。
（赤の条件を満たした時点で黄・青の条件は見ない）

### 用語
- **期限超過タスク** = `タスクの期限 ＜ 今日` かつ `タスク未完了`
- **期限が3日以内のタスク** = `今日 ≦ タスクの期限 ≦ 今日 + 3日` かつ `タスク未完了`

---

## 4. 具体例

### 例① 順調な案件 → 🔵 青
- 案件の完了予定日: 2026-05-31
- 全タスクが予定通り（遅れなし）
- 期限間近の未完タスクも特になし

→ **🔵 青**（予定通り完了できそう）

### 例② タスクが1個遅れている → 🟡 黄
- 案件の完了予定日: 2026-05-31
- 5/8期限のタスクが1個、未完のまま今日（5/9）を迎えている

→ **🟡 黄**（条件 c に該当：期限超過タスク1件）

### 例③ 期限が迫っているタスクが多い → 🟡 黄
- 案件の完了予定日: 2026-05-31
- 期限超過タスクは0件
- でも 5/10〜5/12 に期限のタスクが3個未完で残っている

→ **🟡 黄**（条件 d に該当：3日以内期限の未完タスクが2件以上）

### 例④ タスクが何個も遅れている → 🔴 赤
- 5/3, 5/5, 5/7 期限のタスクがいずれも未完

→ **🔴 赤**（条件 b に該当：期限超過タスクが2件以上）

### 例⑤ 案件全体の完了予定日が過ぎている → 🔴 赤
- 案件の完了予定日: 2026-05-05
- 今日は 2026-05-09（4日超過）

→ **🔴 赤**（条件 a に該当：完了予定日超過）

## 5. KPIとの対応

ボード上部の7つのKPIは以下のように計算される：

| KPI | 計算ロジック |
|---|---|
| 担当件数 | このスコープ（チーム / 個人）の当月案件数の合計 |
| 青件数 | 上記のうち、フラグが青の案件数 |
| 黄色件数 | フラグが黄の案件数 |
| 赤件数 | フラグが赤の案件数 |
| 業完対象 | 当月において業務完了する予定の案件数 |
| 完了割合 | 月初〜本日の完了件数 / 当月完了予定件数 |
| サイクル | 当月完了案件の (完了日 − 受注日) 平均（カ月） |

---

## 6. しきい値の調整について

以下の数値はチューニング可能（運用しながら調整する想定）：

| パラメータ | 現在値 | 意味 |
|---|---|---|
| 赤判定の期限超過タスク数 | **2件以上** | これ以上遅れがあると赤 |
| 黄判定の期限超過タスク数 | **1件以上** | これ以上で黄 |
| 黄判定の期限間近タスク数 | **2件以上** | 期限3日以内タスクがこれ以上で黄 |
| 期限間近の判定日数 | **3日以内** | 何日以内のタスクを「期限間近」と見なすか |

運用してみて「青が出すぎる」「赤が多すぎる」など感覚と合わない場合は、この値を変更して再評価する。
