### 2-2. Supabase Storage セットアップ（ユーザーに実行してもらう SQL）

```sql
-- avatars バケット作成
INSERT INTO storage.buckets (id, name, public)
VALUES ('avatars', 'avatars', true)
ON CONFLICT (id) DO NOTHING;

-- 認証ユーザーは誰でもアップロード可（セキュリティはユルめ方針）
DROP POLICY IF EXISTS "avatars_authenticated_upload" ON storage.objects;
CREATE POLICY "avatars_authenticated_upload" ON storage.objects
  FOR INSERT TO authenticated
  WITH CHECK (bucket_id = 'avatars');

DROP POLICY IF EXISTS "avatars_authenticated_update" ON storage.objects;
CREATE POLICY "avatars_authenticated_update" ON storage.objects
  FOR UPDATE TO authenticated
  USING (bucket_id = 'avatars');

DROP POLICY IF EXISTS "avatars_authenticated_delete" ON storage.objects;
CREATE POLICY "avatars_authenticated_delete" ON storage.objects
  FOR DELETE TO authenticated
  USING (bucket_id = 'avatars');

-- 公開読み取り
DROP POLICY IF EXISTS "avatars_public_read" ON storage.objects;
CREATE POLICY "avatars_public_read" ON storage.objects
  FOR SELECT USING (bucket_id = 'avatars');
```

## 3.5 将来の Phase C：戸籍PDFアノテーション機能

### 3.5-1. 構想

「届いた戸籍をPDFで取り込んで、被相続人を **緑枠**・相続人を **黄枠** でPDF上に囲み、『いつからの戸籍か』という期間情報を付与できる機能」

→ 戸籍書類のレビュー作業を電子化。誰がどの戸籍に該当する／しないかを視覚的に明示できる。

### 3.5-2. 技術アプローチ

| 構成要素 | 採用ライブラリ | 役割 |
|---|---|---|
| PDF表示 | **react-pdf**（pdfjs-dist） | ブラウザ内でPDFを画像レンダリング |
| アノテーションUI | 自前 | PDFの上に重ねた `<div>` でドラッグ矩形作成 |
| 永続化 | Supabase（DB＋Storage） | PDF本体は Storage、矩形データは DBにJSONで |
| PDF書き出し（任意） | **pdf-lib** | 矩形を焼き込んだ新PDFを生成（ダウンロード用） |

全て無料、Next.js + Supabase との相性◎。

### 3.5-3. データモデル（migration 031 で追加予定）

```sql
-- 戸籍ドキュメント本体（documents テーブルとは別建て）
CREATE TABLE koseki_documents (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  case_id UUID NOT NULL REFERENCES cases(id) ON DELETE CASCADE,
  file_path TEXT NOT NULL,          -- Supabase Storage のキー（例: 'koseki/<case_id>/<uuid>.pdf'）
  file_name TEXT NOT NULL,
  koseki_type TEXT,                 -- '戸籍' / '除籍' / '原戸籍' / '改製原戸籍' / '住民票' / 'その他'
  valid_from DATE,                  -- いつからの戸籍
  valid_to DATE,                    -- いつまでの戸籍
  uploaded_by UUID REFERENCES members(id),
  notes TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);
CREATE INDEX idx_koseki_docs_case ON koseki_documents(case_id);

-- ページ単位のアノテーション
CREATE TABLE koseki_annotations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  document_id UUID NOT NULL REFERENCES koseki_documents(id) ON DELETE CASCADE,
  page_index INT NOT NULL,          -- 0始まり
  -- 矩形（PDFページ幅を1.0としたパーセント値で保存。解像度依存を避ける）
  x REAL NOT NULL,
  y REAL NOT NULL,
  width REAL NOT NULL,
  height REAL NOT NULL,
  kind TEXT NOT NULL CHECK (kind IN ('deceased', 'heir')),  -- 緑=被相続人, 黄=相続人
  heir_id UUID REFERENCES heirs(id) ON DELETE SET NULL,     -- 相続人と紐付け（任意）
  label TEXT,                       -- 表示ラベル（手入力）
  notes TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);
CREATE INDEX idx_koseki_anno_doc ON koseki_annotations(document_id, page_index);

-- Supabase Storage: koseki バケット
-- 認証ユーザーは upload/read/delete 可、パブリック読みなし（業務情報なので非公開）
```

### 3.5-5. アノテーション操作仕様

- **作成**：ツールバーで色（被相続人=緑 / 相続人=黄）と紐付ける相続人を選ぶ → PDFページ上でドラッグして矩形を描く → 保存
- **編集**：矩形をクリック → 紐付け相続人やラベルを変更 → 保存
- **削除**：矩形をクリック → ✕ で削除
- **ページ間移動**：◀▶ ボタン + キーボード（左右矢印）
- **倍率変更**：ピンチズーム or Ctrl+ホイール（実装は任意）
- **未保存変更**：自動セーブ（onChange でデバウンス500ms）

### 3.5-6. PDF焼き込みダウンロード（オプション）

`pdf-lib` で：
1. 元PDFをStorageから取得
2. `koseki_annotations` の矩形を全ページに描画（緑/黄の半透明矩形 + ラベルテキスト）
3. 新PDFをブラウザでダウンロード（ファイル名: `戸籍_注釈付き_<案件番号>.pdf`）

### 3.5-7. 既存テーブルとの関係

- `documents` テーブルは引き続き「AI生成書類」「アップロード書類」用。**戸籍は専用テーブル** に分離する方が見通しが良い。
- 案件詳細の **書類タブ** から戸籍タブへの導線を作る、または **戸籍専用タブを新設**。

## 6. 注意点・既知の癖

- **Tailwind 4 の `@theme inline`** で brand 色を定義済み。CSS 変更後は `.next/` キャッシュクリアが必要なケースがあった。

- **localhost dev 起動時**：worktree の `.env.local` がない場合、メインフォルダからコピーが必要。
