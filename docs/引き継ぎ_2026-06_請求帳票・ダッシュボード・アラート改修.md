## 1. このセッションでやったこと（コミット順・新しい順）

### アラート・UI
- **アラートセンター（全ページ共通）**：サイドバーのベルを「アラートセンター」に格上げ。
  - `GET /api/alerts`（`src/app/api/alerts/route.ts`）でログインユーザーのアラートを**ライブ計算**（ロール別・優先度順）。
  - 種別：クレーム / タスク期限超過 / アサイン未完了 / 前受金未入金 / 完了予定日超過 / 週次報告漏れ / 面談メモ未記載 / お客様回答予定日 超過・間近 / 進捗確認依頼。
  - `NotificationBell.tsx` がライブアラート＋既存通知を統合表示。件数バッジ＝アラート＋未読通知。クリックで該当へ。
  - 型・色は `src/lib/alerts.ts`。
- **受注内容 手続区分**：「手続一式」と他手続きを**排他選択**（OverviewTab のローカル `InlineMultiSelect` に `exclusiveValue` 追加）。
- **「案件管理」→「案件一覧」** に改称（サイドバー・/cases 見出し・案件詳細パンくず）。

### 請求・帳票（行/司対応・インボイス）
- **フェーズA**（059）：`invoices.firm_type` 追加。請求書発行モーダルに「発行法人（行/司）」選択（契約形態から既定・連名は別々発行）。色 行=青/司=赤。
- **フェーズB**：請求・入金ページ（BillingClient）に行/司別の集計＋色分け。
- **請求タブを案件ベースに刷新**：`src/lib/billingCaseRows.ts` ＋ `BillingCaseTable.tsx` 新設。
  当月の「受託(受注)/当月完了予定の対応中/当月業務完了の完了」案件を抽出し、区分に応じた請求書状況を1行に集約。
  - チーム別進捗管理の「請求状況」タブを置換（サマリ/メンバー切替/月切替は請求タブでは非表示）。
  - 管理担当マイページに「請求」タブを新設（管理案件一覧の右）。
- **請求タブのサマリ・列統一**：両請求タブ（/billing と 進捗ダッシュ請求タブ）を統一。
  列＝契約形態色｜案件番号｜案件名｜請求分類｜案件ステータス｜受注担当｜管理担当｜入金ステータス｜請求金額｜入金済額｜差額｜請求日｜請求書PDF。
  行=青/司=赤、担当を受注/管理で分割、担当アイコンは `UserAvatar` に統一。
- **帳票プレビュー（`InvoicePreviewClient.tsx`）**：
  - C1：発行元（行/司）を `src/lib/officeProfiles.ts` の `OFFICE_PROFILES[firm_type]` から差し込み（法人名・登録番号・住所・代表者・振込先口座）。
  - C2：`?doc=receipt` で**領収書モード**（タイトル/文言/領収日/領収額に切替・振込先非表示）。請求・入金詳細に「領収書を表示/DL」導線。
  - C3：確定請求書＋立替実費がある場合、**立替実費明細書のA4シートを同梱**（印刷時改ページ）。
  - C1b（060）：立替実費の**課税/非課税**（発行モーダルにセレクタ・費目から自動推定＝官公署手数料/印紙は非課税）。確定請求の**前受金控除**（前受金請求書合計を自動控除）。確定請求書/領収書を**報酬／前受金控除／立替(課税・非課税)／10%対象額／内消費税／請求額**のインボイス準拠表示。
- 「いきいきライフ協会」を**契約形態の選択肢から除外**（相続案件の契約形態は 行政書士法人単独/司法書士法人単独/行・司連名 の3つ。いきいきは終活サービスの別法人で、`officeProfiles` の ikiiki は戸籍/固定資産の遺言執行バリエーション用に残置）。

### ダッシュボード・一覧
- **チームタスク**：下部常時表示をやめ、見出し直下の赤「チームタスク」ボタン→**インライン展開**（`TeamTaskButton.tsx`）。SystemTaskList に `caseAssignees`（受注担当/管理担当列）追加。
- **チーム別進捗管理**：進捗タブの期間/ステータス絞り込みを撤去し**対応中のみ・完了予定日昇順**。「進捗」タブ名→「**管理案件一覧**」。`ProgressCaseTable` に 受注担当/管理担当/受注内容 列追加（管理担当ダッシュボードと共用）。
- **管理案件一覧（MyPageCasesTab）**：受注担当の右に 管理担当/受注内容/完了予定日 を追加。ステータス列は削除（全て対応中のため）、ヘッダー折り返し解消。
- **進捗管理ボード（ProgressKpis）**：フラグ件数を**全アクティブ案件**で集計（月フィルタ除外）、マイページは**対応中のみ**に scope（`computeProgressKpis` に activeStatuses 引数）。フラグ色の説明文を実装（最終接触からの経過日数）に一致。サマリ説明文の誤記修正。
- **面談数を「面談実施日（meeting_executed_date）」ベース**に変更（旧：ステータス遷移依存）。**新規受注の判定を「→受注」遷移**に修正（旧：面談設定済→受注のみ）。`computeSalesMetricsForDay`/`sumSalesMetrics`/`computeSalesMetricsForRange` 追加。
- **受注担当ダッシュボード**：本日に「本日成績」テーブル、年度累計に「当期各月マトリクス」表（横スクロール）。**部全体ダッシュボード本日**にも本日テーブル。月見出しを `YYYY/M`（`monthHeaderLabel`）に統一。
- **相談案件一覧（ConsultationCasesTable）**：「残り日数」列（回答待ちステータスのみ）、ステータス絞り込みの選択肢修正、期間切替を「本日/当月/当期累計」に。タブ名「当月面談」→「相談案件一覧」。
- 案件ステータス「不受注」→「**不受託**」改称、案件ヘッダーのフローに「紹介のみ」追加。
- **レイアウト**：`(authenticated)/layout.tsx` の `<main>` に `min-w-0`（幅広テーブルをページ全体に広げずページ内スクロールに閉じ込め）。

## 2. 重要な設計メモ・注意点

- **発行法人マスタ** `src/lib/officeProfiles.ts`：行/司/いきいきの法人情報・インボイス登録番号・振込口座を保持。帳票はここを参照。
- **帳票テンプレ原本**：`docs/請求書・領収書.xlsx`（請求書 前受金/確定、領収書 前受金/確定、立替実費 各 行/司）。行/司の差は**発行元ブロックのみ**（＝ OFFICE_PROFILES と一致）。本文レイアウトは同一。
- **内消費税の計算**は「報酬＋課税立替を税込みとみなし ×10/110」の内税方式。**前受金の消費税の扱い等は経理確認推奨**（ズレたら計算式調整）。
- **面談数=面談実施日ベース**に変えたので、運用で面談実施日の入力が前提。
- 立替実費を**請求モーダル以外**から追加した場合、課税区分は既定 null（表示上は課税扱い）。

## 3. 未決・要確認の宿題

1. **「管理担当をアサインする」タスクの担当者**（`sys_assign_manager`）：現状 `assign_role='sales'`（受注担当）。本来は「管理担当グループ内の割り振り担当者」が行う作業。**割り振り担当者をシステム上どう特定するか未決**（特定メンバー指定 / 管理担当全員 / チームタスク欄のみ表示）。決まれば migration で `generate_case_alert_tasks` の該当部分を修正。
2. **帳票の税計算**を経理に確認（特に前受金の消費税）。
3. **アラートの閾値・種別**は初期値。運用を見て調整（例：前受金未入金の経過日数、回答予定日「間近」の日数=現状+2日）。
4. **アラート要約をマイページ見出し右にも出す**（二重に気づける）案：未実装。希望あれば追加。
5. 立替実費の課税区分の**自動推定マッピング**（官公署手数料=非課税）は叩き台。費目ごとの正否は運用で上書き前提。
