## 残作業

### 1. PeriodSwitcher の集計ロジック対応
現状 UI のみ配置。実際の period 値（today/month/ytd/by_month）で集計範囲を切り替える実装が必要。

対象ファイル: `/dashboard/team/[teamId]/page.tsx` ほか
- `searchParams.period` を読む
- `today` ベースの集計を `month` / `ytd` / `by_month` でも計算できるよう拡張
- `computeSalesDailyMetrics` / `computeSalesMetrics` を period 対応に
- `by_month` の場合は月別棒グラフ風の表示が必要（新規コンポーネント）

### 2. 管理担当ダッシュボード新規
画像の構成:
- 部署全体 → 既存 `/dashboard/dept` を改修して期間切替対応
- 管理担当 全体 + チーム別 → `/dashboard/team/[teamId]/manager` のような新規ページが必要
- 主KPI: **請求完了件数**（invoices.status='入金済' の件数 / 期間）と**案件数**
- 受注担当ダッシュボードと同じく PeriodSwitcher + TeamMemberTabs + メンバー別テーブル

### 3. 案件タブの担当者列分離
画像の指示:
> 担当者というタブの列は、受注担当と、管理担当を分けて、両方記載した

対象:
- `/cases` 一覧画面（既に分かれているか確認）
- `/my?tab=cases`（MyPageCasesTab は現状担当者列なし、追加）
- `/dashboard/team/[teamId]/progress` の案件テーブル

### 4. メンバー（受注担当以外）のタブ
画像の管理担当ダッシュボードでは、管理担当メンバーが並ぶ。
TeamMemberTabs の primary_role フィルタを切替可能にする必要あり。
- `/dashboard/team/[teamId]` → primary_role='sales' のみ
- `/dashboard/team/[teamId]/manager` → primary_role='manager' のみ

### 5. 部署全体ダッシュボードの期間切替対応
`/dashboard/dept` も同じく PeriodSwitcher 対応。

### 6. アクセス制御（任意）
- マイページは現状「全認証ユーザー」が自分のページを見られる
- 部署全体は管理職のみ閲覧可、などのアクセス制御を追加することも検討
