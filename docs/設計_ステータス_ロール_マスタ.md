# ステータス制御・ロール設定・マスタデータ設計書

## 1. ロール定義

### 1.1 ロール一覧

| ロール | 説明 | 案件での役割 |
|--------|------|-------------|
| **受注担当** | 営業マン。案件の獲得・依頼者との窓口 | 案件を受注し、面談・依頼者対応を行う |
| **管理担当** | 進捗管理・タスク割振り・監督 | 案件全体の進捗を管理し、タスクの割振りと品質管理を行う |
| **管理担当アシスタント** | タスク消化の実行者 | 割り振られたタスクを実行・処理する |
| **LP担当** | リスティング/LP経由の問合せ対応 | LP経由のリード対応 |
| **経理担当** | 請求・入金・精算管理 | 立替実費計算、請求書作成、入金確認を行う |

### 1.2 権限マトリクス

| 操作 | 受注担当 | 管理担当 | アシスタント | LP担当 | 経理担当 |
|------|:-------:|:-------:|:-----------:|:------:|:-------:|
| **案件ステータス変更** | ○ | ◎ | × | × | × |
| **タスク一括生成** | × | ◎ | × | × | × |
| **タスク担当者割振り** | × | ◎ | × | × | × |
| **タスクステータス変更（自分担当）** | ○ | ◎ | ○ | × | ○ |
| **タスクステータス変更（他人担当）** | × | ◎ | × | × | × |
| **Wチェック実施（完了/差戻し判定）** | ○ | ◎ | ○ | × | ○ |
| **タスク手動追加** | ○ | ◎ | ○ | × | × |
| **書類生成** | × | ◎ | ○ | × | × |
| **請求・精算操作** | × | ○ | × | × | ◎ |
| **メンバー/マスタ編集** | × | ◎ | × | × | × |
| **案件閲覧** | ○ | ◎ | ○ | ○ | ○ |

◎ = 主担当操作（必須権限）、○ = 可能、× = 不可

## 2. ステータス制御

### 2.1 案件ステータス遷移図

```
架電案件化 → 面談設定済 → 検討中 → 受注 → 対応中 → 完了
                ↓                       ↓
              失注                   保留・長期 → 対応中（復帰）
```

| 遷移元 | 遷移先 | 許可ロール | 備考 |
|--------|--------|-----------|------|
| 架電案件化 | 面談設定済 | 受注担当, 管理担当 | 面談日程確定時 |
| 面談設定済 | 検討中 | 受注担当, 管理担当 | 面談実施後 |
| 面談設定済 | 失注 | 受注担当, 管理担当 | キャンセル |
| 検討中 | 受注 | 受注担当, 管理担当 | 契約締結時 |
| 検討中 | 失注 | 受注担当, 管理担当 | 不成約 |
| 受注 | 対応中 | 管理担当 | タスク生成・着手時 |
| 対応中 | 保留・長期 | 管理担当 | 長期中断 |
| 対応中 | 完了 | 管理担当 | 全タスク完了＋納品完了 |
| 保留・長期 | 対応中 | 管理担当 | 再開時 |

### 2.2 タスクステータス遷移図

```
未着手 → 対応中 → Wチェック待ち → 完了
                      ↓
                    差戻し → 対応中（修正後再提出）
```

| 遷移元 | 遷移先 | 許可ロール | 備考 |
|--------|--------|-----------|------|
| 未着手 | 対応中 | 主担当, 副担当, 管理担当 | タスク着手時 |
| 対応中 | Wチェック待ち | 主担当, 副担当 | 作業完了→チェック依頼 |
| 対応中 | 未着手 | 主担当, 管理担当 | 着手取消 |
| Wチェック待ち | 完了 | **主担当以外の任意のメンバー** | Wチェック通過 |
| Wチェック待ち | 差戻し | **主担当以外の任意のメンバー** | Wチェックで問題あり |
| 差戻し | 対応中 | 主担当, 副担当 | 修正して再提出 |

**Wチェックのルール**:
- Wチェック実施者 = 主担当以外の誰でも可（自分の作業を自分でチェックしない）
- Wチェック完了時にチェック実施者名を記録する
- 差戻し時はコメント必須（差戻し理由）

## 3. Supabaseテーブル設計

### 3.1 テーブル一覧

| テーブル名 | 用途 | 種別 |
|-----------|------|------|
| `members` | メンバーマスタ | マスタ |
| `roles` | ロール定義 | マスタ |
| `member_roles` | メンバー×ロール紐付け | マスタ |
| `role_permissions` | ロール×権限マトリクス | マスタ |
| `task_templates` | タスクテンプレート（38件） | マスタ |
| `phases` | フェーズ定義 | マスタ |
| `status_definitions` | ステータス定義（案件・タスク） | マスタ |
| `status_transitions` | ステータス遷移ルール | マスタ |
| `cases` | 案件データ | トランザクション |
| `tasks` | タスクデータ | トランザクション |
| `task_assignees` | タスク×担当者（主/副） | トランザクション |
| `documents` | 書類データ | トランザクション |
| `case_members` | 案件×担当者（受注/管理/アシスタント等） | トランザクション |
| `task_comments` | タスクコメント・差戻し理由 | トランザクション |
| `activity_log` | 操作ログ（ステータス変更等） | トランザクション |

### 3.2 主要テーブル定義

#### members（メンバーマスタ）
```sql
CREATE TABLE members (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  email TEXT UNIQUE,
  avatar_color TEXT DEFAULT '#6B7280',
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

#### roles（ロール定義）
```sql
CREATE TABLE roles (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  key TEXT UNIQUE NOT NULL,        -- 'sales', 'manager', 'assistant', 'lp', 'accounting'
  label TEXT NOT NULL,             -- '受注担当', '管理担当', ...
  description TEXT,
  sort_order INT DEFAULT 0
);
```

#### member_roles（メンバー×ロール）
```sql
CREATE TABLE member_roles (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  member_id UUID REFERENCES members(id),
  role_id UUID REFERENCES roles(id),
  UNIQUE(member_id, role_id)
);
```

#### role_permissions（権限マトリクス）
```sql
CREATE TABLE role_permissions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  role_id UUID REFERENCES roles(id),
  permission TEXT NOT NULL,        -- 'case.status.change', 'task.bulk_generate', 'task.assign', ...
  allowed BOOLEAN DEFAULT false,
  UNIQUE(role_id, permission)
);
```

#### task_templates（タスクテンプレート）
```sql
CREATE TABLE task_templates (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  key TEXT UNIQUE NOT NULL,        -- 'koseki_create', 'koseki_mail', ...
  label TEXT NOT NULL,             -- '戸籍請求書作成'
  phase TEXT NOT NULL,             -- 'Phase1:相続人調査'
  category TEXT NOT NULL,          -- '戸籍'
  procedure_text TEXT,             -- 作業手順テキスト
  is_manual BOOLEAN DEFAULT false, -- 手動起票タスクかどうか
  sort_order INT DEFAULT 0,
  is_active BOOLEAN DEFAULT true
);
```

#### status_definitions（ステータス定義）
```sql
CREATE TABLE status_definitions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  type TEXT NOT NULL,              -- 'case' or 'task'
  key TEXT NOT NULL,               -- '未着手', '対応中', etc.
  label TEXT NOT NULL,
  color TEXT,
  sort_order INT DEFAULT 0,
  UNIQUE(type, key)
);
```

#### status_transitions（遷移ルール）
```sql
CREATE TABLE status_transitions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  type TEXT NOT NULL,              -- 'case' or 'task'
  from_status TEXT NOT NULL,
  to_status TEXT NOT NULL,
  allowed_roles TEXT[],            -- {'manager','sales'} etc.
  requires_comment BOOLEAN DEFAULT false,  -- 差戻し時はtrue
  UNIQUE(type, from_status, to_status)
);
```

#### cases（案件データ）
```sql
CREATE TABLE cases (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  case_id TEXT UNIQUE NOT NULL,    -- 'R7-A00127'
  deal_name TEXT NOT NULL,
  status TEXT DEFAULT '架電案件化',
  client_name TEXT,
  deceased_name TEXT,
  date_of_death DATE,
  order_date DATE,
  completion_date DATE,
  difficulty TEXT,                  -- '易','普','難'
  procedure_type TEXT[],           -- {'手続一式','登記'}
  additional_services TEXT[],      -- {'相続税申告','不動産売却'}
  tax_filing_required TEXT,        -- '要','不要','確認中'
  tax_filing_deadline DATE,
  property_rank TEXT,              -- 'S','A','B','C'
  total_asset_estimate BIGINT,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

#### case_members（案件×担当者）
```sql
CREATE TABLE case_members (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  case_id UUID REFERENCES cases(id),
  member_id UUID REFERENCES members(id),
  role TEXT NOT NULL,               -- 'sales', 'manager', 'assistant', 'lp'
  UNIQUE(case_id, member_id, role)
);
```

#### tasks（タスクデータ）
```sql
CREATE TABLE tasks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  case_id UUID REFERENCES cases(id),
  template_key TEXT,                -- TASK_TEMPLATESのkeyと紐付け（手動追加はNULL）
  title TEXT NOT NULL,
  phase TEXT NOT NULL,
  category TEXT,
  status TEXT DEFAULT '未着手',
  priority TEXT DEFAULT '通常',     -- '通常','急ぎ','外出タスク'
  due_date DATE,
  procedure_text TEXT,
  wcheck_by UUID REFERENCES members(id),  -- Wチェック実施者
  wcheck_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

#### task_assignees（タスク×担当者）
```sql
CREATE TABLE task_assignees (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  task_id UUID REFERENCES tasks(id) ON DELETE CASCADE,
  member_id UUID REFERENCES members(id),
  role TEXT NOT NULL DEFAULT 'primary',  -- 'primary' or 'sub'
  UNIQUE(task_id, member_id)
);
```

#### task_comments（コメント・差戻し理由）
```sql
CREATE TABLE task_comments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  task_id UUID REFERENCES tasks(id) ON DELETE CASCADE,
  author_id UUID REFERENCES members(id),
  content TEXT NOT NULL,
  comment_type TEXT DEFAULT 'comment', -- 'comment', 'rejection', 'wcheck_pass'
  created_at TIMESTAMPTZ DEFAULT now()
);
```

### 3.3 RLS（Row Level Security）方針

| テーブル | ポリシー |
|---------|---------|
| マスタ系 | 全員SELECT可。INSERT/UPDATE/DELETEは管理担当のみ |
| cases | SELECT: 案件の担当者（case_members）のみ。UPDATE: 権限に応じて |
| tasks | SELECT: 案件の担当者のみ。UPDATE: 権限に応じて |
| task_assignees | SELECT: 案件の担当者のみ。INSERT/DELETE: 管理担当のみ |

## 4. 移行ステップ（参考）

1. **Phase A**: Supabaseプロジェクト作成 → テーブル作成 → マスタデータ投入
2. **Phase B**: フロントエンドからSupabase APIに接続 → マスタデータ読み込み
3. **Phase C**: トランザクションデータ（案件・タスク）のCRUD実装
4. **Phase D**: ステータス遷移ルール＋権限チェックをEdge Functionで実装
5. **Phase E**: RLS設定＋認証（Supabase Auth）連携
