# 入金消込の精度向上 ― 検討資料

作成: 2026-06-20

## 1. 現状と課題

銀行入金CSV（みずほ・きらぼし）を取り込み、未入金の請求へ突合している。
精度を下げている構造的な要因は以下。

- **案件番号の記入率が約3割。** 振込時に案件番号を入れるよう依頼しているが、高齢の依頼者が多く実際には入らない。
- **振込名義人が依頼者と違う。** 配偶者・子などが代理で振り込むケースが多い。
- **ノイズが混ざる。** 依頼者以外からの別目的入金、利息・振替などが明細に含まれる。
- 結果として、**金額だけが頼り**になる場面が多い。

## 2. すでに実装済みのシステム改善（前提）

抜本策の前に、ロジック側は次まで対応済み（commit `81897e2`）。

- 突合キーを **「振込人カナ＋金額」** のマスターキー方式に変更。
- 依頼者に **振込名義人（カナ）** を登録（代理振込は代理人カナを登録）。半角カナ・ひらがなを全角カタカナに正規化して照合。
- 取引区分を **振込のみ** に限定し、利息・振替・出金などのノイズを除外。
- 案件番号（旧番号含む）が摘要にあれば最優先の手掛かりに使用。

これで「記入されていれば自動確定／カナが登録されていれば自動確定」までは到達する。
ただし **記入率3割・カナ未登録** という入口の問題は、システムだけでは埋まらない。
そこで業務側の抜本策を2案で比較する。

---

## 3. 抜本策（2案）

入口（振込が起きる時点）で**一意に特定できる情報を持たせる**のが本質。方向性は2つ。

### 案A：バーチャル口座（入金専用口座）

案件ごと（または顧客ごと）に**異なる振込先口座番号**を銀行から払い出し、請求書に記載する。
入金は口座番号で特定できるため、**振込名義人も案件番号の記入も一切不要**。

- **長所**：ほぼ100%自動消込。記入ミス・代理振込・同名・同額衝突の影響をすべて受けない。消込が構造的に「悩まなくなる」。
- **短所**：銀行サービスの契約・初期設定・月額/件数コストが発生。口座番号の払い出し管理を新システムに組み込む必要がある。みずほ・きらぼし双方に同等サービスがあるか要確認。

### 案B：ユニーク金額方式

請求額の**下数桁を案件由来の端数**にして、金額自体を一意化する（例：報酬100,000円 → 100,007円）。
同額衝突がなくなり、**金額だけで自動確定**できる。

- **長所**：銀行契約不要・低コストで即日導入可能。既存の突合ロジックにそのまま乗る。
- **短所**：端数分（数円〜数十円）が請求額に乗る違和感、説明の手間。端数の採番ルール管理が必要。端数が偶然一致する別入金の可能性はゼロではない（カナ併用で吸収）。

### 比較

| 観点 | 案A バーチャル口座 | 案B ユニーク金額方式 |
|---|---|---|
| 自動消込の精度 | ◎ ほぼ100% | ○ 高い（カナ併用で補強） |
| 導入コスト | △ 銀行契約・月額・開発 | ◎ ほぼ無料・即日 |
| 記入率・代理振込の影響 | 受けない | 受けない |
| 依頼者への説明負担 | 小（口座が変わるだけ） | 中（端数の理由） |
| 実装規模 | 大（口座払い出し管理） | 小（採番のみ） |

---

## 4. 推奨：段階導入

1. **まず案B（ユニーク金額方式）を低コストで導入。**
   既存ロジック（カナ＋金額）と相性が良く、すぐ精度が上がる。端数の採番を新システムで自動付与する。
2. **将来、件数・運用負荷が増えたら案A（バーチャル口座）へ。**
   消込を構造的にゼロ手間にしたい段階で銀行サービスを契約し、口座払い出しを組み込む。

> いずれの案でも、**振込名義人カナの登録**を受任時の運用として定着させると、案B導入前の現時点でも自動確定率が上がる。
