## 2. 直近セッションで完了したこと（タスク詳細関連）

### Migration 043: tasks に列追加（要 Supabase 適用）
```sql
ALTER TABLE tasks
  ADD COLUMN IF NOT EXISTS expected_completion_date DATE,
  ADD COLUMN IF NOT EXISTS completed_at DATE;

-- 完了に切り替わった瞬間に自動で当日日付をセット
CREATE OR REPLACE FUNCTION set_task_completed_at() RETURNS TRIGGER AS $$
BEGIN
  IF NEW.status = '完了' AND (OLD.status IS DISTINCT FROM '完了') AND NEW.completed_at IS NULL THEN
    NEW.completed_at = CURRENT_DATE;
  END IF;
  IF NEW.status <> '完了' AND OLD.status = '完了' THEN
    NEW.completed_at = NULL;
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

DROP TRIGGER IF EXISTS trg_set_task_completed_at ON tasks;
CREATE TRIGGER trg_set_task_completed_at
  BEFORE UPDATE ON tasks FOR EACH ROW EXECUTE FUNCTION set_task_completed_at();
```
→ `supabase/migrations/043_tasks_completion_dates.sql`

### 3-1. 前段作業の確認セクション
案件詳細のタスク詳細画面で、**前のタスクの結果を読み取って表示** + 評価入力を行うセクション。

**仕様**:
- 前のタスクへのリンク
- 前段作業の実施結果（前タスクの「実施結果」フィールドを読み取り表示）
- 作業実施者（前タスクの started_by_member の名前）
- 作業完了日（前タスクの completed_at）
- 内容評価ラジオ: ○（不備なし）/ ×（差戻し）
- 不備内容（差戻しの場合の理由、textarea）

**必要なDB変更**:
- tasks に列追加:
  - `prev_task_evaluation TEXT CHECK IN ('不備なし', '差戻し')`
  - `prev_task_defect_note TEXT`
- または既存 ext_data JSON に格納（追加 migration 不要）

**実装方針**:
- TaskDetailClient で前タスク（`prereqDeps[0].from_task` あたり）の情報を取得
- 評価フォームを左カラムに追加（基本情報の下）
- 評価=「差戻し」を保存したら、前タスクのステータスを「差戻し」に変える連携も検討

### 3-2. 差戻しステータス + 通知（ベルマーク）
**仕様**:
- TASK_STATUSES に「差戻し」を追加（既に存在する場合は確認）
- 前タスクで「差戻し」評価が付くと、その前タスクのステータスが自動で「差戻し」になる
- 差戻されたタスクの担当者にシステム内通知（ベルマーク）
- タスク一覧でもステータス「差戻し」を視認できる

**必要なDB変更**:
- 通知テーブルを新規作成:
  ```sql
  CREATE TABLE notifications (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    member_id UUID NOT NULL REFERENCES members(id) ON DELETE CASCADE,
    type TEXT NOT NULL,         -- 'task_returned' / 'task_assigned' 等
    case_id UUID REFERENCES cases(id) ON DELETE CASCADE,
    task_id UUID REFERENCES tasks(id) ON DELETE CASCADE,
    title TEXT NOT NULL,
    body TEXT,
    is_read BOOLEAN DEFAULT false,
    created_at TIMESTAMPTZ DEFAULT now()
  );
  CREATE INDEX idx_notifications_member ON notifications(member_id, is_read, created_at DESC);
  ```
- RLS: ログイン中ユーザーは自分宛て通知のみ閲覧

**実装方針**:
- サイドバー（または右上のアバター付近）にベルアイコン + 未読件数バッジ
- クリックで通知一覧ドロップダウン
- タスク詳細の評価フォームで「差戻し」保存時:
  1. 前タスクのステータスを「差戻し」に更新
  2. 前タスクの started_by （or 担当者）に notification を insert
