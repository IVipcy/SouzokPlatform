## 2. 本セッションで完了した主な変更

### 2-1. タスク詳細画面の大改造（多数の commit）

- **3カラムレイアウト**: 左 (前タスク紐づけ + 前段作業の確認) / 中央 (基本情報・作業内容) / 右 (次タスク紐づけ + タイムライン)

- **前段作業の確認セクション**: 前タスクの実施結果を読み取り表示 + ○ 不備なし / × 差戻し 評価
- **差戻し**: status='差戻し' を独立ステータスとして復活、評価×で前タスクを自動的に差戻しに変更
- **次タスク紐づけ**: チェックボックス1リスト + 「新しいタスクを作成」フォーム + 「+既存から選ぶ」と分離した2段構成
- **0件時の表示**: 🚩スタート地点 / 🏁ゴール地点
- **タスク分類フィールド**: 基本情報に追加（事務管理/管理担当/受注担当/経理担当、`work_role` を流用）
- **作成物セクション**: 案件全体の書類をクロスタスクで表示
- **ヘッダー**: タスク名:/案件名:/手続き区分: のラベル付き表記

## 6. 注意点・落とし穴

### 6-1. 'use client' ファイルからの named export
**サーバコンポーネントから 'use client' モジュールの named export を import すると Server Components Render Error**

- 純粋関数は `src/lib/...` の通常モジュール (no 'use client') に置く
- 例: `parsePeriod` は `src/lib/dashboardPeriod.ts` に分離してある
- TypeScript の型チェックでは検出されないので注意

### 6-2. サーバ → クライアントへの prop は serializable のみ
**関数 prop を Server から Client へ渡すと Render Error**

- 例: `TeamMemberTabs` の `buildMemberHref` を関数 → `basePath: string` に変更済み

### 6-3. 既存テーブルが不在のときのフォールバック
新規 migration 未適用環境でも動くよう、`dashboard_team_members` などの SELECT は try/catch で隔離。
受注担当チーム別では fallback として `members.team_id === teamId` で代替。

### 6-4. 進捗管理ボードは既存実装
`/dashboard/team/[teamId]/progress/page.tsx` には既に下記が実装済。重複実装しないこと:
- 8つのKPI（担当件数 / 青 / 黄 / 赤 / 紫 / 完了割合 / サイクル / 請求件数）
- メンバー切替 (TeamMemberNav)
- 進捗 / 請求状況 タブ (ProgressViewTabs)
- 月選択 (MonthSelector)
- フラグ別案件テーブル (ProgressCaseTable)
- 請求状況ビュー (BillingStatusView)
