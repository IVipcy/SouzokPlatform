# AI書類作成 定義（2エクセル精読まとめ）

対象エクセル:
- `docs/作成書類一覧（契約書・委任状・請求書・領収書）.xlsx`（37シート）
- `docs/作成書類一覧（戸籍請求書・名寄帳評価証明）.xlsx`（11シート）

各テンプレは、案件共通の変数を **「入力元」シート**に集約し、各書類セルが `=入力元!◯◯` で参照する作り。
事務所の署名欄（住所・代表者・TEL/FAX・口座）は**各テンプレに直書き**され、契約形態/書類で出し分く。
※ サンプル/参照データ（▲▲▲、横浜太郎、`←委任状の方の住所`、`↓触らないで`、`経理用` 等）は**提出時に消す前提**＝生成時に除去する。

---

## 1. 入力元（案件共通変数）→ 案件詳細データソース対応

「入力元」シート（C4:AJ14）に定義された変数。各テンプレはこれを参照する。

| 入力元 変数 | セル | 案件詳細のデータソース | 備考 |
|---|---|---|---|
| 管理番号 | L4/N4/P4/R4/T4 | `cases.case_number` | 区切り分割（例 2606LP0001 → R/7/-/Y/0 のように分割表示） |
| 郵便番号 | L5,Q5 | `clients.postal_code` | 〠・- は固定装飾 |
| 依頼者住所 | L6 | `clients.address` | |
| マンション名以下 | L7 | （空欄） | **分割しない方針**。`clients.address`全文を住所行(L6)に入れ、建物名行(L7)は空欄。将来きっちり2行化したい場合のみ`building_name`列を追加 |
| 依頼者氏名 | L8 | `clients.name` | |
| 外字（依頼者） | AC9 有/無 | 外字フラグ（依頼者） | 有なら「依頼者氏名に外字あり」を注記生成 |
| 生年月日 | L9(元号)/P9/T9/X9 | `clients.birthday` | 元号・年・月・日に分割 |
| 被相続人氏名 | L13 | `cases.deceased_name` | |
| 外字（被相続人） | AC13 有/無 | 外字フラグ（被相続人） | 有なら「・被相続人氏名に外字あり」を注記生成 |
| 死亡日 | L14(令和)/P14/T14/X14 | `cases.date_of_death` | 元号・年・月・日に分割 |

外字注記の式（全テンプレ共通）:
`=入力元!AJ9 & 入力元!AJ14 & IF(OR(外字依頼者="有",外字被相続人="有"),"に外字あり","")`

## 4. FMT選択ロジック（UI: 書類を選ぶ→FMTを選ぶ→生成）

- **契約書**: `契約形態`（行政単独/司法単独/行・司連名）× `受注内容`（相続一式/遺言/執行/贈与信託/1%）× `財産調査の有無` で1テンプレに確定。
- **委任状**: `当事者`（行のみ/司のみ/連名）× `業務`（相続/登記/遺言/贈与信託/法定相続情報/預金解約あり/遺言検索）で確定。
- **請求書/領収書**: `発行主体`（行/司）× `種別`（前受金/確定/立替実費）。発行主体は契約形態に連動（連名時はどちらで請求するか選択）。
- **戸籍/名寄帳**: `名義`（行政/司法）。

→ **契約形態（cases.contract_type）＋受注区分（service_category）＋業務** から推奨FMTを**初期選択**してモーダルを開き「推奨」バッジを表示。ユーザーは別FMTに自由変更して確定できる（提案は初期値、強制しない）。

### 押印（ハンコ画像）の配置
- 司法書士法人＝添付1枚目、行政書士法人＝添付2枚目。法人に応じた画像をセル幅に合わせて配置。
- 配置箇所: 契約書=受任者（乙=行政／丙=司法）署名欄／委任状=代理人（行政・司法）氏名欄／請求書・領収書=発行者（代表社員）右の押印枠。
- 委任者本人の「印」欄（委任状 X11 等）は空欄のまま（お客様が押す）。

## 5. 主なフィールド → データソース対応（書類別）

共通（全書類）: §1 の入力元変数（管理番号・依頼者氏名/住所/生年月日・被相続人氏名/死亡日・外字）。
署名欄: §2 の事務所マスタ（契約形態/発行主体で出し分け）。

| 書類 | 固有フィールド | ソース |
|---|---|---|
| 契約書 | 報酬額 / 報酬基準（別紙 or 金額） | `cases.fee_*`、契約・報酬・請求タブ |
| 契約書 | 委任業務の範囲・契約期間 | 受注内容（service_category）から定型文 |
| 委任状 | 委任者=相続人（氏名/住所/生年月日） | `clients`（依頼者＝相続人） |
| 委任状 | 被相続人・死亡日 | `cases.deceased_name` / `date_of_death` |
| 請求書 | 件名・請求区分・請求額・内消費税 | 契約・報酬・請求タブ（前受金/確定）、`invoices` |
| 請求書 | 発行年月日 | 発行時の日付 |
| 領収書 | 請求書の内容を引用（件名/区分/額）・領収日 | 対応する請求書 |
| 戸籍請求書 | 本籍・筆頭者・請求に係る者・使用目的・同封小為替 | 相続人調査（戸籍請求 koseki_requests）、被相続人=deceased_name |
| 戸籍請求書 | 請求者（委任者）の住所・氏名 | `clients`（委任状の依頼者） |
| 名寄帳・評価証明 | 所有者（故 氏名）/所在地番/家屋番号/区分 | 財産調査（real_estate_properties） |
| 名寄帳・評価証明 | 申請内容（名寄帳/評価/非課税・年度・部数） | 手入力 or 既定 |

## 6. 参照データ（生成時に除去すべきもの）

提出版では消す。生成物には**印刷範囲内の確定値だけ**を残し、印刷枠外のヘルパー類は**すべて削除**する。
- 計算用セル/参照マスタ: `=VLOOKUP`/`=IF` 等の補助セル、`AB4:AX5` 等の事務所ルックアップ表、戸籍テンプレの `W9:Y12`、請求書の `経理用`/`税抜価格確認用` の `AC列`、`BC列` のメモ等
- サンプル氏名/住所: `▲▲▲▲`、`○○ ○○`、`横浜 太郎`、`横浜 華子`、`鈴木 よし江`、`故 佐野 行廣` 等
- 案内・操作メモ: `←委任状の方の住所`、`←委任者の氏名記入`、`↓触らないで`、`↓○使ってね`、`←入力してください`、`←どっちか選ぶ`、`↓【…場合】例：…`
- 既定の日付 `=TODAY()` は生成日（または指定日）に確定。

## 7. 実装メモ（次のAI書類作成ブラッシュアップ）

- テンプレxlsxは `public/templates/<doc>/<variant>.xlsx` に配置（戸籍請求書/固定資産は実装済の方式を踏襲）。
- 生成API（ExcelJS）: 入力元変数を案件/依頼者から流し込み、署名欄は契約形態/発行主体でテンプレ選択 or マスタ差し込み。参照データ（§6）は空に。
- 署名マスタ（§2）はコード定数 or DBに集約し、テンプレ間で重複させない（住所変更時の修正を1箇所に）。
- 生成物は `documents` テーブル＋Storage に保存し、「作成書類一覧」に表示（実装済）。タスク詳細から生成時は `task_id` を付与。
- **実装済**（2026-06-18）: 委任状10様式（`/api/documents/ininjo`）／契約書4様式（`/api/documents/keiyaku`）／請求書・領収書（前受金）行・司（`/api/documents/invoice`）／確定請求書＋立替実費明細を1ファイル2シートで出力 行・司（`/api/documents/kakutei`。報酬・立替は税込入力で内消費税自動計算、前受金は消費税対象外で差引）。
- **未実装**: 契約書（1％）・契約書（執行）＝構造が大きく異なり入力アンカーも無いため別途／戸籍・名寄帳（戸籍/固定資産は既存実装）。
- **対象外**（2026-06-18 確定）: 簡易契約書（単独/行司）、いきいきライフ協会、LM復代理。
- **確定事項**（2026-06-18）: 住所は分割しない（全文を住所行へ）／FMTは推奨を初期選択し最終はユーザー確定／押印は司法=`public/templates/stamps/shiho.png`・行政=`public/templates/stamps/gyosei.png`（配置済）／印刷枠外のヘルパー・参照マスタ・注意書きは全削除。
